# Pico2 向けのFMP3ベンチマークプログラム

## コンフィギュレーション
- シングルプロセッサ向けのベンチマークに関してはFMP3のコンフィギュレーションをマルチコアとシングルコアから選択可能
- トップの ConfigSingle.cmake から選択可能
- 1行目を有効にするとシングルコア，2行目を有効にするとマルチコアのコンフィギュレーションでFMP3をビルドする．
  
```
set(TNUM_PRCID 1)
# set(TNUM_PRCID 2)
```

## 計測方法
- 2種類の計測方法をサポートしている
  - 内蔵タイマーによる計測
    - 1us精度による計測
  - GPIOによる計測
    - GP2/GP3/GP4による計測
    - 使用するポートは./fmp3/target/rp2350-arm-s_pico_sdk/taget_test.h のマクロで指定する

```
#define M_GPIO_NO1  2
#define M_GPIO_NO2  3
#define M_GPIO_NO3  4
```

- 切り替え方法
  - ./perf/perf.h でマクロの定義で切り替える
  - 内蔵タイマーによる計測
    - USE_HISTOGRAM を定義する
  - GPIOによる計測
    - USE_HISTOGRAM を未定義にする

## 使用方法
- Pico拡張をインストールしたVSCodeで各フォルダを開いてビルド

## ロジック・アナライザを用いたデータの取得
Saleae のロジック・アナライザを用いて，エクセルの頻度グラフを生成する方法について説明する．

- 計測を実施
- チャネル毎にcsvにエクスポートする
- csvファイルをエクセルファイルにする
- 「=(A4-A3)*1000000000」をC3に入力し，以降の列Cに適用する（計測結果をnsで示すため）
- B3の「1」を右クリックし，「フィルター」→「選択したセルの値でフィルター」を選択
- C3以降をコピーし，D3以降にペーストする
- B3の「1」を右クリックし，「フィルター」→「"channel 0"からフィルターをクリア」を選択
- 「=ROUNDDOWN(D3,0)」をE3に入力し，以降の列Eに適用する
- E3からE2002までをコピーし，F3以降に値をペーストする
- F3からF2002を選択し，「データ」→「重複の削除」→「現在選択されている範囲を並べ替える」→「重複の削除」→列Fが選択されていることを確認して「OK」をクリックする
- 列Fの残った値を選択し，右クリックから「並べ替え」→「昇順」→「現在選択されている範囲を並べ替える」→「並べ替え」をクリックする
- 「=COUNTIF($E$3:$E$20002,F3)」をG3に入力し，以降の（列Fに値がある分まで）列Gに適用する
- 列Fと列Gの値を選択し，縦棒グラフを作成する
